# 경사로 가이드

경사로는 `<` 와 `>` 키를 눌러야 하는 계단과 달리 해당 방향으로 걷기만 해도 수직 이동이 가능한 지형입니다.

## 플래그 의미

- `GOES_UP` 과 `GOES_DOWN` 은 계단식 연결입니다. `<`, `>` 키를 통한 수직 이동과 계단 탐색에 사용됩니다.
- `RAMP_UP` 과 `RAMP_DOWN` 은 진입 시 발동하는 연결입니다. 플래그가 붙은 타일로 들어가면 대상이 즉시 수직으로 1층 이동합니다.
- `RAMP` 는 타일을 일반 이동 도우미, 등반 판정, 일부 경로 탐색에서 경사로처럼 취급하도록 표시합니다. 하지만 이것만으로 `RAMP_UP` 이나 `RAMP_DOWN` 과 같은 효과가 생기지는 않습니다.
- `RAMP_END` 는 낮은 계단 형태 경사로의 정렬된 끝을 표시하며, `avatar_action::ramp_move` 가 더 자연스럽게 오르내리도록 하는 데 사용됩니다.

## 경사로 동작 방식

- `RAMP_UP` 타일에 들어가면 생물이나 차량은 같은 `x,y` 좌표의 `z + 1` 로 이동합니다.
- `RAMP_DOWN` 타일에 들어가면 생물이나 차량은 같은 `x,y` 좌표의 `z - 1` 로 이동합니다.
- 게임은 맵의 다른 위치에서 짝이 되는 타일을 찾지 않습니다. 경사로 전환은 엄격하게 같은 열(`x,y`)에서만 일어납니다.

관련 코드 경로:

- `src/avatar_action.cpp` 는 플레이어가 `RAMP_UP` / `RAMP_DOWN` 으로 들어갈 때 이동을 조정합니다.
- `src/monmove.cpp` 는 몬스터 이동에도 같은 규칙을 적용합니다.
- `src/vehicle_move.cpp` 는 차량 중심과 precalc 지점에도 같은 규칙을 적용합니다.
- `src/pathfinding.cpp` 는 경사로 전환을 같은 열의 z-레벨 변화로 캐시합니다.

## 짝 맞추기 규칙

일반적인 2층 경사로에서는 타일을 같은 `x,y` 에서 수직으로 짝이 맞아야 합니다.

- 아래 z-레벨의 높은 쪽 끝: `RAMP_UP`
- 위 z-레벨의 낮은 쪽 끝: `RAMP_DOWN`

기본 도로 경사로 지형도 이 방식을 사용합니다.

- 아래 z-레벨의 `t_ramp_up_high`
- 그 바로 위, 위 z-레벨의 `t_ramp_down_low`

반대 방향도 같은 쌍을 거꾸로 사용합니다.

## 단일 타일 커스텀 경사로

시체 경사면(`zlope`) 같은 단일 타일 경사로는 특히 주의가 필요합니다.

- 타일에 `RAMP_UP` 이 있으면, 그 타일로 들어갈 때 이동체를 같은 `x,y` 의 `z + 1` 에 놓으려고 합니다.
- 따라서 커스텀 상승 경사로는 윗타일이 실제로 그 전환을 받을 수 있을 때만 만들어야 합니다.
- 실제로는 지붕이나 천장 아래에 단일 타일 상승 경사로를 생성하면 안 됩니다. `zlope` 같은 생성형 경사면에서는 위 타일이 통과 가능해야 하고 `NO_FLOOR` 를 가져야 합니다.

일반 바닥 타일 아래에 `RAMP_UP` 타일을 두면 의도한 기하와 맞지 않는 결과가 나옵니다. 경사로는 위로 올라간다고 주장하지만, 도착하는 열은 여전히 위에서 막혀 있기 때문입니다.

## 각 접근 방식을 써야 할 때

- 계단, 사다리, 맨홀처럼 의도적으로 수직 이동을 해야 하는 경우에는 `GOES_UP` / `GOES_DOWN` 을 사용합니다.
- 일반 이동의 일부로 z-레벨이 바뀌어야 하는 타일에는 `RAMP_UP` / `RAMP_DOWN` 을 사용합니다.
- 등반 도우미나 일반 경사로 로직에서도 경사로처럼 동작해야 한다면 `RAMP` 를 추가합니다.

## 실전 체크리스트

- 가능하면 상승 경사로와 하강 경사로의 끝점을 같은 `x,y` 에서 수직으로 짝지으십시오.
- `RAMP` 만으로 자동 z-레벨 전환이 생긴다고 생각하면 안 됩니다.
- 지붕 아래에는 단일 타일 상승 경사로를 두지 마십시오.
- 플레이어만 쓰는 명시적 수직 상호작용이 필요하다면, 경사로 대신 계단이나 사다리를 사용하십시오.
