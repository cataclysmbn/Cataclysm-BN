# スロープガイド

スロープは、`<` と `>` に割り当てられた明示的な階段操作を使わずに、歩行可能な z レベル遷移を作るための mapgen 側の仕組みです。

## フラグの意味

- `GOES_UP` と `GOES_DOWN` は階段式のリンクです。明示的な垂直移動と階段探索で使われます。
- `RAMP_UP` と `RAMP_DOWN` は進入時に発動するリンクです。フラグ付きタイルへ移動すると、移動体は直ちに 1 z レベル移動します。
- `RAMP` は、そのタイルを一般的な移動補助、登攀判定、一部の経路探索でスロープ状として扱う印です。ただし、これだけで `RAMP_UP` や `RAMP_DOWN` と同じ効果にはなりません。
- `RAMP_END` は低い階段状スロープの揃った端を示し、`avatar_action::ramp_move` がぎこちなくならずに登れるように使われます。

## 遷移の適用方法

- `RAMP_UP` タイルに入ると、生物や車両は同じ `x,y` の `z + 1` へ移動します。
- `RAMP_DOWN` タイルに入ると、生物や車両は同じ `x,y` の `z - 1` へ移動します。
- ゲームはマップ上の別の場所から対応タイルを探しません。スロープ遷移は厳密に同じ列(`x,y`)でのみ行われます。

関連するコード経路:

- `src/avatar_action.cpp` は、プレイヤーが `RAMP_UP` / `RAMP_DOWN` に入った際の移動を調整します。
- `src/monmove.cpp` は、モンスター移動にも同じ規則を適用します。
- `src/vehicle_move.cpp` は、車両の中心点と precalc 点にも同じ規則を適用します。
- `src/pathfinding.cpp` は、スロープ遷移を同じ列での z レベル変化としてキャッシュします。

## 対応付けの規則

通常の 2 層スロープでは、タイルは同じ `x,y` で垂直に対応付ける必要があります。

- 下の z レベルの高い端: `RAMP_UP`
- 上の z レベルの低い端: `RAMP_DOWN`

組み込みの道路スロープ地形もこの方式を使います。

- 下の z レベルの `t_ramp_up_high`
- その真上、上の z レベルの `t_ramp_down_low`

逆方向でも同じ組を逆向きに使います。

## 1 タイルのカスタムスロープ

死体斜面のような 1 タイルスロープでは、追加の注意が必要です。

- タイルが `RAMP_UP` を持つ場合、そのタイルに入ると移動体は同じ `x,y` の `z + 1` に配置されます。
- そのため、カスタムの上りスロープは、上側のタイルが実際にその遷移を受け入れられる場所にだけ作るべきです。
- 実際には、屋根や天井の下に 1 タイル上りスロープを生成してはいけません。`zlope` のような生成斜面では、上のタイルは通行可能であり、`NO_FLOOR` を持つべきです。

通常の床タイルの下に `RAMP_UP` タイルを置くと、意図した形状と食い違う結果になります。スロープは上へ登ると示しているのに、到達先の列は上から塞がれたままだからです。

## どの方式を使うべきか

- 階段、はしご、マンホールなど、意図的な垂直操作には `GOES_UP` / `GOES_DOWN` を使います。
- 通常移動の一部として z レベルを変えたいタイルには `RAMP_UP` / `RAMP_DOWN` を使います。
- 登攀補助や一般的なスロープロジックでもスロープのように振る舞わせたい場合は `RAMP` を追加します。

## 実用チェックリスト

- 可能なら、上りスロープと下りスロープの端点は同じ `x,y` で垂直に対応付けてください。
- `RAMP` だけで自動 z レベル遷移が起きると思わないでください。
- 屋根の下に 1 タイル上りスロープを置かないでください。
- プレイヤー専用の明示的な垂直操作が必要なら、スロープではなく階段やはしごを使ってください。
