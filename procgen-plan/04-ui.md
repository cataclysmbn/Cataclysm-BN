# UI

## Browser

The craft list must stay short.

Show:

- `Sandwich`
- `Stew`
- `Knife spear`

Do not show every filled variant.

## Flow

`recipe list -> builder -> preview -> finalize`

Normal recipes stay on the old path.

Proc recipes branch into a builder.

## Builder layout

- left: slots
- mid: candidates for the current slot
- right: live preview

## Preview rules

Preview is hot path.

It should use:

- slot counts
- cheap stat sums
- prebuilt candidate facts
- cached tag/allergen bitsets

It should not use:

- full item recreation
- deep recursive component scans
- Lua on every keypress

## Candidate index

Build once when opening the builder.

Indexes:

- `role -> items`
- `tag -> items`
- `slot -> items`

Optional cache key:

- `schema + inv revision`

## Builder state

The builder should keep small, flat state:

- slot order
- candidate fact indexes per slot
- chosen fact indexes per slot
- fast preview blob
- selection cursor / scroll

Do not keep copied child item trees in the builder state.

Only finalize should materialize full result state.

## Preview content

Food:

- name
- kcal
- vitamins
- allergens
- tags / traits
- mass / vol
- freshness source

Gear:

- name
- bash / cut / pierce / to-hit
- moves
- dur
- mats
- repair note

## UI subtask cut

Split work into small slices:

1. route proc recipes to a stub builder
2. slot list panel
3. candidate panel
4. preview panel
5. input handling
6. cache / scroll polish
